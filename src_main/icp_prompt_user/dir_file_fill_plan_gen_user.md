# 文件级实现规划生成

请你根据以下信息生成清晰的文件级编程实现规划。

**用户的原始编程需求：**

USER_ORIGINAL_REQUIREMENTS_PLACEHOLDER

**精炼后的结构化编程需求：**

```json
REFINED_REQUIREMENTS_PLACEHOLDER
```

**各个文件的功能描述：**

```json
DIR_FILE_CONTENT_PLACEHOLDER
```

请结合以上信息，生成一个清晰完整的文件级实现规划，说明：

1. **程序如何初始化和启动**
   - 主入口文件是什么
   - 初始化时需要做哪些操作
   - 如何准备程序运行环境

2. **文件的调用顺序和层级关系**
   - 哪些文件是基础层（被其他文件依赖）
   - 哪些文件是业务层（实现核心功能）
   - 哪些文件是应用层（协调整体流程）
   - 各个文件如何被调用，调用的顺序是什么

3. **数据/实例/结构的流转过程**
   - 关键的数据结构在哪些文件中被定义
   - 实例对象在哪些文件中被创建
   - 这些数据/实例如何在文件间传递和使用

4. **功能实现的具体路径**
   - 用户需求中的每个功能如何通过组织多个文件来实现
   - 各个文件在实现某个功能时的具体分工和协作方式

5. **整体架构如何达成用户目标**
   - 从宏观角度总结所有文件如何协同工作
   - 说明这个架构设计如何完整实现用户的原始编程需求

请严格按照系统提示词中定义的格式输出结果。

禁止添加任何解释性文字，直接输出规划内容。
