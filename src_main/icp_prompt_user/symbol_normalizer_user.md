# 符号命名规范化

请你根据以下信息，将从IBC文件AST中提取的自然语言符号转换为符合编程规范的标识符，并判断其合理的可见性作用域。

## 文件信息

**目标编程语言：** TARGET_LANGUAGE_PLACEHOLDER

**文件路径：** FILE_PATH_PLACEHOLDER

**文件上下文：** CONTEXT_INFO_PLACEHOLDER

## 待规范化的符号列表

以下是从当前文件的AST中提取的自然语言符号，每个符号包含其类型和描述信息：

AST_SYMBOLS_PLACEHOLDER

## 任务要求

请为上述每个符号提供：

1. **规范化的标识符名称**（normalized_name）
   - 必须符合标准编程语言标识符规范
   - 字母或下划线开头
   - 仅包含字母、数字、下划线
   - 根据目标编程语言的命名习惯进行标识符规范化
   - 保留原始符号的核心语义
   - 避免使用编程语言保留关键字

2. **可见性作用域**（visibility）
   - 必须从以下预定义值中选择一个：
     - `private`: 仅在定义所在类内部可见
     - `protected`: 在定义类及其子类中可见
     - `public`: 类的公共成员，外部可访问
     - `module_local`: 在当前模块目录内可见
     - `global`: 整个工程范围内全局可见
   - 根据符号的描述和用途判断
   - 遵循最小可见性原则


**注意事项：**
- 确保输出的是有效的JSON格式
- 外层key必须与输入的自然语言符号名完全一致
- 每个符号都必须有对应的输出
- normalized_name 必须符合目标语言的标识符规范
- visibility 必须是预定义的枚举值之一
- 不要包含任何额外的字段或注释
